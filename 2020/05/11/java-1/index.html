Java-1.String的length方法使用需要谨慎！！

String.length方法

码点

码点指的就是编码表中的某个字符对应的代码值。

代码单元

16个二进制位称为一个代码单元

17个代码平面

java中使用的默认编码集是utf-16
当一个码点位于第一代码平面：基本多语言平面时，只需要一个代码单元就可以实现存储，而当码点是U+10000-U+10FFFF时需要两个代码平面才能实现存储，U+0000-U+D7FF用于表示第一代码平面的代码单元，U+D800-U+FFFF用于表示后面16个代码平面的代码单元，U+D800-U+DBFF 用于表示第一代码单元，共十位；U+DC00-U+DFFF 用于表示第二代码单元，共十位，在表示辅助字符的20位时，4位表示代码平面，16位表示一个平面中的字符

String的底层实现

由于java的跨平台性，使用的统一的默认编码可以使得在不同电脑编辑的文件，在不同电脑上运行的结果是一样的。
上面讲这么多看起来和和length方法一点关系没有的话是因为，这可以帮助你们理解接下来的知识点。
String方法底层使用的是char[]存储的，java中char的大小是固定的，一个char是四个字节，也就是16位，和第一代码平面(基本多语言平面)中字符的大小是一样的，这就使得在使用常用的字符时，一个char就可以装入一个字符，非常方便。
但有一个问题需要注意，length方法，这个方法返回的是string字符串对象中char集合的长度，这就意味着当你在初始化String时，使用了第一代码平面(基本多语言平面)之外的字符，String就需要用两个char来进行存储。
那么问题来了如何获得正确的字符数呢

这个方法返回的是字符串中码点的个数，实际的底层实现就是判断每一个char中代码平面的二进制值大小，在U+0000-U+D7FF之间就是一个char表示一个字符，大于就是两个char表示一个字符。
在使用中要注意哦！！
