openAPI#

常用的注解#

  • @OpenAPIDefinition：用于定义API的全局信息，如标题、描述、版本、许可证等。
  • @Operation：用于定义单个操作的信息，如摘要、描述、标签、响应等。
  • @Parameter：用于定义操作的参数，如名称、描述、类型、是否必需等。
  • @RequestBody：用于定义操作的请求体，如内容类型、模式等。
  • @ApiResponse：用于定义操作的响应，如状态码、描述、内容等。
  • @Schema：用于定义模型的属性，如名称、描述、类型等。

        info = @Info(
                title = "Custom API title",

                url = "https://baidu.com"
        ),
        extensions = @Extension(

                properties = {
                        @ExtensionProperty(name = "key1", value = "value1"),

                        url = "http://localhost:8080",
                        description = "Local server"
                ),
                @Server(
                        url = "https://example.com/api/v1",
                        description = "Production server"
                )
        }
)

  • info：用于展示代码页的信息如：标题、版本号、联系邮件等

  • tags：打标签，用于对不同的请求进行分类（在OpenAPIDefinition中只是罗列具有的类型，具体接口所属的类型，需要在接口方法上使用@Tag注解）

  • externalDocs：可以添加想要的链接

  • extensions：提供关键字，还没用明白

  • servers：可以提供全局请求的地址信息

@Operation#

  • 作用：配置方法信息

  • 使用位置：使用在接口的方法上

    1
2
3
4
5

    @GetMapping("/hello")
@Operation(summary = "Say hello", description = "Returns a greeting message", tags = {"greeting"})
public String hello() {
  	return "Hello";
}

  • summary：简要介绍

  • description：详细介绍

  • tags：所属的类型

  • method：用于指定操作的HTTP方法，如GET、POST等。这个属性在springdoc-openapi中不是必需的，因为springdoc-openapi会根据Spring Boot项目中的@RequestMapping或@GetMapping等注解来自动推断操作的HTTP方法。但是，如果您想要覆盖默认的HTTP方法，您可以使用@Operation注解中的method属性。

  • requestBody：用于定义操作的请求体，如内容类型、模式等。这个属性在springdoc-openapi中不是必需的，因为springdoc-openapi会根据Spring Boot项目中的@RequestBody注解来自动推断操作的请求体。但是，如果您想要覆盖默认的请求体，或者添加一些额外的信息，您可以使用@Operation注解中的requestBody属性。例如：

    @RestController
@RequestMapping("/api")
public class HelloController {

    @PostMapping("/hello")
    @Operation(summary = "Say hello", description = "Returns a greeting message", tags = {"greeting"},
        requestBody = @io.swagger.v3.oas.annotations.parameters.RequestBody(
            description = "The name of the person to greet",
            content = @Content(
                mediaType = "application/json",
                schema = @Schema(implementation = String.class)
            ),
            required = true
        )
    )
    public String hello(@RequestBody String name) {
        return "Hello, " + name;
    }
}

@Parameter#

@Parameter注解有多种用途，具体取决于它所在的上下文。一般来说，@Parameter注解是用于定义方法参数的元数据，如名称、描述、类型、约束等1。例如：

@RestController
@RequestMapping("/api")
public class HelloController {

    @GetMapping("/hello/{name}")
    @Operation(summary = "Say hello", description = "Returns a greeting message", tags = {"greeting"})
    public String hello(@PathVariable @Parameter(description = "The name of the person to greet") String name) {
        return "Hello, " + name;
    }
}

@RequestBody#

@RequestBody注解是用于将HttpRequest的body映射到一个Java对象，实现自动反序列化。这个注解通常用于接收JSON或XML格式的请求体。例如：

1
2
3
4
5
6
7
8

@RestController
@RequestMapping("/api")
public class HelloController {
    @PostMapping("/hello")
    public String hello(@RequestBody User user) {
        return "Hello, " + user.getName();
    }
}

在这个例子中，@RequestBody注解是用于将请求体中的JSON数据转换为User对象。

@Schema#

@Schema注解是用于定义数据模型的元数据，如名称、描述、类型、约束等。这个注解通常用于配合@RequestBody或@ApiResponse等注解来描述请求体或响应体的结构。例如：

1
2
3
4
5
6
7
8
9

@RestController
@RequestMapping("/api")
public class HelloController {
    @PostMapping("/hello")
    @Operation(summary = "Say hello", description = "Returns a greeting message", tags = {"greeting"})
    public String hello(@RequestBody @Parameter(description = "The user to greet") @Schema(implementation = User.class) User user) {
        return "Hello, " + user.getName();
    }
}

在这个例子中，@Schema注解是用于指定请求体的实现类为User类。

@ApiResponse#

@ApiResponse注解是用于定义操作的响应信息，如状态码、描述、内容等。这个注解通常用于配合@Operation或@ApiResponses等注解来描述操作可能返回的不同响应。例如：

@RestController
@RequestMapping("/api")
public class HelloController {
    @PostMapping("/hello")
    @Operation(summary = "Say hello", description = "Returns a greeting message", tags = {"greeting"},
        responses = {
            @ApiResponse(responseCode = "200", description = "Successful operation"),
            @ApiResponse(responseCode = "400", description = "Invalid input"),
            @ApiResponse(responseCode = "500", description = "Internal server error")
        }
    )
    public String hello(@RequestBody User user) {
        return "Hello, " + user.getName();
    }
}

在这个例子中，@ApiResponse注解是用于定义操作可能返回的三种状态码和对应的描述。
